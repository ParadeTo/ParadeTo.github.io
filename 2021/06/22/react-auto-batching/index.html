总结

本文介绍了 Legacy 模式下 React 17 更新批处理的实现方式，以及在某些场景下失效的原因，对比了 React 18 Concurrent 模式下自动批处理的相关知识。简单来说，Legacy 模式是通过某个变量来标识是否处于批处理的状态，而 Concurrent 模式是通过判断 root 上是否已调度了任务以及更新优先级是否变化来决定是否开启新的更新任务还是复用已调度的任务。
